Does GDP Explain Flu Rates?¶

As we explore the factors which impact flu rates worldwide, the per capita GDP of a country, as well as its total GDP may be important indicators. Wealthier countries may have lower rates of infection.

In this excercise, we'll use simple linear models and Pearson's r to test correlation between GDP and flu rates. The latter lets us understand quickly if two factors have a linear correlation with one another. This can be useful information when deciding to whether or not linear regression models are appropriate.

Once we've connected to our database, we want to pull data for per capita GDP as well as flu rates. However, our flu data is in total cases per week, so we'll need to compute a per capita infection rate for each country as well. We'll do this by adding up each countries infections and dividing by the countries population.

%%sql select g.percapgdp as per_cap_gdp, sum(f.TOTAL_A_INFECTIONS/p.total_pop) as per_cap_a, 
sum(f.total_b_infections/p.total_pop) as per_cap_b, f.country 
from flu_statistics f, population_info p, gdp_data g
where f.country=p.country
and g.country=f.country
and f.year=p.year
and g.year=f.year
and g.percapgdp is not null
group by f.country, g.percapgdp

The scatterplot shows what might be a slight trend, but it also looks a lot like a flat line. Checking correlation using Pearson's r may let us know if there's really any relationship there.

The first value returned the correlation slope: 0 would be no correlation, 1 would be completely positive linear correlation, and -1 would be an inverse correlation. The second value is the confidence interval on the correlation. It's very small, which suggests the correlation is important enough to model.

Since both correlations seem significant, we'll build a regression model of type-A infections to per capita GDP. This single-factor regression attempts to find a function of the form: y=mx+b to the data. y is our type-A data and x is the GDP data.

In [9]: 
per_cap_data = add_constant(per_cap_data)
gdp_model = ols("per_cap_a ~ per_cap_gdp", data=per_cap_data).fit()
print(gdp_model.summary())

Examining the table above, we can see that the p-value (P>|t|) for per capita GDP is statistically significant. However, the coefficent value is very small, as are the R-squared and adjusted R-squared values. This suggests that the GDP has an effect on type-A infection rates, but it doesn't completely explain the data. There are likely other factors at work.

Along with the diagnostic statistics, we should also consider the residuals of the model. That is, the difference between what our model predicts and what the actual observed data is. A component-component and residual plot can show us how well the model fits the data. In this case, it fits rather well, but the effect is not tremendous.

Summary¶

In this exercise, we used linear modeling and Pearson's correlation coefficient to examine the effect of GDP on flu rates worldwide. It appears that a country's wealth has statistically significant, but overall minor effect on flu rates. We got to this conclusion by:

  • Using SQL to extract data into Pandas
  • Viewing scatterplots of the data
  • Using Pearson's r to determine if there's a linear correlation
  • Building simple regression models to determine the type of effect
