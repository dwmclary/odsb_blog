Does Living in Cities Influence Flu Rates?¶

Along with GDP and other factors, the amount of a country's population which is either urban or rural may impact flu transmission rates. For example, do more densely-packed countries with high urban population percentages transmit the disease more easily than largely agrarian countries?

In this exercise, we'll examine the effect of rural and urban population on flu rates using basic linear regression modeling. Specifically, we'll use ordinary least squares to quickly model the impacts of these factors on a response variable: per capita flu rates.

As usual, we import the necessary libraries and connect to our database.

Population¶

As with GDP and other factors, we really want to focus on how flu rates are impacted per capita. We don't want factors like total population to cloud our ability to analyze the data. As such, we'll need to extract the data from the database in a fashion that gives us per capita rates for type-A and type-B flu infection. This is very simple: we sum the total infections for each country and divide them by the population of each country.

In [3]: 
%%sql select p.urban_pop, p.rural_pop, sum(f.TOTAL_A_INFECTIONS/p.total_pop) as per_cap_a, 
sum(f.total_b_infections/p.total_pop) as per_cap_b, f.country 
from flu_statistics f, population_info p
where f.country=p.country
and f.year=p.year
group by f.country, p.urban_pop, p.rural_pop
order by urban_pop

There appears to be at least some indication that increased urban populations are associated with higher per capita flu rates. Let's build a linear model and see if the relationship is significant.

In [6]: 
urban_flu = add_constant(urban_flu)
urban_model = ols("per_cap_a ~ urban_pop", data=urban_flu).fit()
print(urban_model.summary())

Looking at the results table, we can see that the p-value (P>|t|) for urban_population is significant, and does impact type-A rates. However, R-squared and adjusted R-squared suggest that urban population percentage does not fully explain the data.

A look at the residuals can show us where our model predicts poorly. statsmodels includes the plot_regress_exog function, which will give us a set of residual plots. Notice that as urban population percentage increases, the model explains fewer of the data points.

We can also look at an interaction plot to determine the magnitude of interaction at various levels. Let's break the urban population percentage into quartiles and add it as a new column in our DataFrame.

In [19]: 
def pop_quantiles(p):
    if p < 25:
        return "very low"
    elif p >= 25 and p < 50:
        return "low"
    elif p >= 50 and p < 75:
        return "moderate"
    elif p >= 75 and p < 100:
        return "high"
    else:
        return "very low"
urban_gdp['pop_q'] = pd.Series(map(pop_quantiles, urban_gdp['urban_pop']))

The interaction plot doesn't show us a perfect, linear interaction between GDP and urban population percentage, but we can see that there's a much more pronounced effect from high GDP when the population is highly urban.

We cannot claim to have fully explained the data, but we can say that there is a likely interaction between these factors on per capita type-A rates.
