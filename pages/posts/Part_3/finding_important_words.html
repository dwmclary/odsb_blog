Finding Important Words With Spark¶

In a previous lesson, we used Apache Spark to find news articles which contained the word "flu." This gave us a sense of how many articles might be about the flu, as well as how frequently the word appeared relative to articles like "a," "an," and "the." However, we haven't yet figured out what these news stories are really about. Nor have we figured out how important news about the flu is over time.

In this lesson, we'll do just those things. We'll be using Spark again, through the PySpark interface. This time, though, we'll be doing things more closely related to data science. We'll analyze data using a well-know technique, apply a twist to treat a different problem, and build a reusable dataset from which others can benefit. Specificially, this lesson will cover:

  • How to measure term importance using the TF-IDF transform.
  • How to treat groups of news articles over time
  • How to generate a dataset which can be shared used to build a "news search" notebook

Determining Word Importance: Stemming and TF-IDF¶

First, we're going to revisit the question of word-importance. A previous lesson illustrated that the most frequently used words in English are typically not what news stories are about. Let's load our news data as an RDD and examine a story. It's obvious to us what the story is about, but the number of small, connecting words dwarfs those we care about.

For speed, we'll start by taking a smple of our news data. This is done with the built-in RDD function sample.

To really measure word importance, we're going to need to clean up and normalize the text. This involves a few steps, which we put into the clean_word function.

  • Make all words lowercase
  • Remove special characters and punctuation
  • Stem the words -- i.e., "talk" and "talking" should both be treated as "talk"

Stemming is an important part of working with text. While there are many packages which accomplish this, we'll use a simple python package to simplify things.

With this function in place, we can create two RDDs which may be interesting: the full text and the titles. We'll be focusing on the full text for obvious reasons. We're going to create a special RDD in this case: a PairRDD. This allows us to identify the full text of an article by its title: it's referred to as a key-value pair. The key is the title and the value is the full text.

In order to perform our TF-IDF transformation, we'll also need the total frequency of each word in the data set. This is equivalent to the word-count we performed previously. However, we'll use Spark's countByKey method to provide us a Python dict to work with.

We'll also need some other information for the computation of IDF, or inverse document frequency. That is, we want to scale term frequency (how often a word appears) by how common that word is across all documents. The more unique that word is to the article, the more important it may be.

To do this, we need to compute a few more things:

  • the total number of documents
  • the total number of words in each document

Spark's counting functions make this easy.

Once this is done, we can define a function to compute TF-IDF for each document given a set of dictionaries. The function article_tf_idf does just this.

In [10]: 
def article_tf_idf(article_total, words_per_article, tf_per_article, occ_across_articles):
    result = []
    for key, value in tf_per_article.items():
        article = key[0]
        term = key[1]
        wpm = words_per_article[article]
        ocm = occ_across_articles[term]
        tf_idf = (float(value)/wpm) * np.log(article_total/ocm)
        result.append({"article":article, "term":term, "score":tf_idf})
    return result

Applying the function to our dictionary yields the sort of results we'd expect. Words like "live" and "cup" have higher scores than words like "than" and "have." We've also put the results into a list of dictionaries, which makes integration with pandas and matplotlib simple.

Examining Word Importance Over Time¶

Our TF-IDF function allows us to determine how important words are in each article, which gives us a better idea of what the articles are about. However, what if we want to understand how important a word is to the news over time? Is flu more important in March, 2010 than in April 2011? Can we use TF-IDF to determine that?

In fact, we can, it just requires reframing the problem. Instead of considering words inside of articles, we want to consider words inside of date ranges. If we do that, then our approach can be applied easily.

Let's get started by making a function date_to_key which will help us construct PairRDDs for each month,year combination in the news dataset. Then we can get an RDD documents_over_time which has dates for keys and article text for values.

In [7]: 
def date_to_key(d):
    return tuple(map(lambda x: int(x), d.split("-")[0:2]))
In [8]: 
documents_over_time = news_json.map(lambda x: (date_to_key(x['date']), x['fulltext'])).map(lambda x: (x[0], map(clean_word, x[1].split()))).map(lambda x: (x[0], " ".join(x[1])))

At this point, we have a number of values for each key. That's going to make computing our TF-IDF over time more difficult. What we really want is a bag of words for all article text for a given month. We can use a reducer to do this. We'll just append all the articles for the month together with spaces.

In [9]: 
wordbags_over_time = documents_over_time.reduceByKey(lambda x, y: x + " " + y)

From here, computing TF is just like doing it for an article!

Finally, we make a new function for dealing with our monthly data. The only change is in the dictionary keys of the result.

In [87]: 
def monthly_tf_idf(month_total, words_per_month, tf_per_month, occ_across_months):
    result = []
    for key, value in tf_per_month.items():
        month = key[0]
        term = key[1]
        wpm = words_per_month[month]
        ocm = occ_across_months[term]
        tf_idf = (float(value)/wpm) * np.log(month_total/ocm)
        result.append({"month":month, "term":term, "score":tf_idf})
    return result

import pandas as pd

def to_ascii(v):
    try:
        return str(v)
    except:
        return ""

def flatten_for_data_frame(r):
    y,m= r['month']
    return {"date":pd.datetime(y,m,1),"term":r['term'],"score":r['score']}
    
def flatten_for_write(r):
    r['term'] = to_ascii(r['term'])
    return r
