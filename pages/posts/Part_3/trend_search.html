Building a Trend Search with Big Data and Big Data SQL¶

In previous exercises we've examined the importance of terms over time. However, our investigation has been somewhat cursory -- we only searched for data which had terms like "flu" in it. But does the flu impact the importance of certain terms at certain times?

If we could pair up the term-trend data we have in our Hadoop cluster with the statistics data we have in our database, we could examine that proposition. In this excercise, we'll do exactly that:

  • We'll examine the outbreaks of pathogenic H5N1 flu across time
  • We'll use Oracle Big Data SQL to access data we previously processed with Spark.
  • We'll combine these two to examine how outbreaks may have changed term importance

As usual, we'll need our trusty imports before we start.

Previously, we looked at how to do batch-based copies of data from a database into HDFS, the Hadoop filesystem. Big Data SQL, is somewhat different: it allows the database to declare a table on data in HDFS. It can then be queried without copying or loss of performance.

Big Data SQL requires an external table, so let's declare on of type ORACLE_HDFS on our flu_search data in HDFS.

In [4]: 
%%sql CREATE TABLE flu_search (search_doc VARCHAR2(4000))
  ORGANIZATION EXTERNAL (
  TYPE ORACLE_HDFS
  DEFAULT DIRECTORY DEFAULT_DIR
   LOCATION ('/user/oracle/flu_search/part*')
  ) 
  REJECT LIMIT UNLIMITED

Even if we want to keep the table definition as a single column, a database view allows us to treat the JSON as if it were a table with columns.

In [31]: 
%%sql create or replace view flu_view as
  select
    cast (f.search_doc.term as varchar2(255)) term,
    cast (f.search_doc.score as number) score,
    to_date(f.search_doc.month, 'DD-MM-YYYY') term_date
  from flu_search f

H5N1 is the clear standout here, with 2 really big peaks -- one between 2006 and 2007, and the other just in recent years. Since there doesn't seem to be a correlated rise with the countries which had the most outbreaks, perhaps we need to consider different comparison criteria.

Before that, though, let's take our plotting code and wrap it up in a method. This will make it easier to plot the results of any SQL query on our trend data.

In [37]: 
def plot_term_search(term_search):
    term_search = term_search.DataFrame()
    term_search.index = pd.DatetimeIndex(term_search['term_date'])
    term_search['score'] = term_search['score'].astype(float)
    gt = term_search.groupby("term")
    fig, ax = plt.subplots(nrows=1, ncols=1, figsize=(16,12));
    for t,g in gt:
        g.score.plot(label=t)
    plt.legend(loc="best")

A quick spot check using terms we know should be associated with H5N1 validates our thinking. There were large spikes in the news around "flu" and "outbreak" at the same time as "h5n1." Of course, these terms are likely from the same news stories.

Because most of the terms in our flu search data have been "stemmed," it may be difficult to find exact matches. Instead we'll use the SQL like operator to get near matches for all of the countries of interest. By placing these like comparison in a case statement, we can control the groups returned for plotting.

In [40]: 
%%sql select term_date, term, score from ( 
        select term_date, 
            case
                when term like 'british%' then 'british'
                when term like 'europe%' then 'europe'
                when term like 'viet%' then 'vietnam'
                when term like 'asia%' then 'asia'
                when term like 'turk%' then 'turkey'
                when term like 'ghana%' then 'ghana'
                when term like 'hong%' then 'hong kong'
                when term like 'asia%' then 'asia'
                when term like 'ukraine%' then 'ukraine'
                when term = 'h5n1' then 'h5n1'
                else null
            end as term,
            score
        from flu_view
    ) where term is not null
    order by term_date

Summary¶

In this exercise, we considered how flu outbreaks might impact the importance of words in the news. That is, can our term-importance search data show us what terms might have been associated with outbreaks of H5N1 pathogenic flu. To do this, we used Big Data SQL to expose the HDFS data we created with term importance over time. From there, everything could be done through basic SQL and our plotting interface.
